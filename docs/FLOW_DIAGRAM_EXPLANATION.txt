═══════════════════════════════════════════════════════════════
👤 CITIZEN FLOW (LEFT SIDE - YELLOW)
═══════════════════════════════════════════════════════════════

1. SUBMIT (Image and text)
   ↓
   Citizen opens App UI and submits civic issue
   - Uploads image
   - Enters description
   - Adds location

2. POST Request
   ↓
   App UI sends HTTP POST to backend
   - Endpoint: /api/tweets
   - Data: FormData with image + text

3. Invoke Module
   ↓
   Middleware triggers AI Engine
   - Calls Python scripts
   - Passes image for processing

4. [Content Detected]
   ↓
   NSFW Detection runs
   - Uses NudeNet model
   - Checks if image is appropriate

5. Result UNSAFE → Show Error (if inappropriate)
   OR
   Result SAFE → Continue

6. [Invokes CNN Classifier]
   ↓
   Civic Issue Classification
   - Uses TensorFlow model
   - Predicts category (Pothole, Garbage, etc.)

7. Return Category
   ↓
   AI Engine returns classification result

8. Insert Record
   ↓
   Middleware saves to MongoDB
   - Stores: image, text, category, status=pending

9. Return ID
   ↓
   Database returns unique issue ID

10. 200 OK Success
    ↓
    Citizen receives confirmation
    - Issue ID displayed
    - "Successfully submitted" message

═══════════════════════════════════════════════════════════════
👨‍💼 ADMIN FLOW (RIGHT SIDE - GREEN)
═══════════════════════════════════════════════════════════════

1. View Priority Dashboard
   ↓
   Admin opens Dashboard UI
   - Sees list of pending issues
   - Sorted by priority/date

2. GET Sorted Data
   ↓
   Dashboard sends HTTP GET request
   - Endpoint: /api/tweets?status=pending
   - Fetches pending issues

3. Fetch Pending Issues
   ↓
   Middleware queries MongoDB
   - Filters by status=pending
   - Sorts by priority

4. Return Data
   ↓
   Issues displayed on dashboard
   - Shows: image, description, location, category

5. Mark Resolved
   ↓
   Admin reviews and marks issue as resolved
   - Changes status to "resolved"
   - Adds admin notes

6. PUT Update Status
   ↓
   Dashboard sends HTTP PUT request
   - Endpoint: /api/tweets/:id/status
   - Data: { status: "resolved" }

7. Update Record
   ↓
   Middleware updates MongoDB
   - Changes status field
   - Adds resolvedAt timestamp

8. Confirm Success
   ↓
   Database confirms update
   - Returns updated record

9. Send SMS Notification
   ↓
   System sends SMS to citizen
   - Message: "Your issue has been resolved"
   - Includes issue ID

10. View and Rate
    ↓
    Citizen receives notification
    - Opens app to view resolution
    - Can rate the service

11. POST Feedback
    ↓
    Citizen submits rating
    - Endpoint: /api/feedback
    - Data: { rating, comment }

12. Save Rating
    ↓
    Feedback stored in database
    - Linked to original issue
    - Used for analytics
